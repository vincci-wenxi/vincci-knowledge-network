## 一、这次改了什么（v2.4 → v2.5）

| 维度 | v2.4（旧） | v2.5（新） |
|------|-----------|-----------|
| 原子笔记定义 | 混装 TC/CC/MM | **只放已证实的**（TC/TM/TN） |
| 原创内容 | 混在 03-Atomic 里（CC/MM + is_original） | **独立成 04-原创 库**（OT/OM/OC） |
| 分类第一维 | 形态（术语/概念/模型） | **已证实 vs 原创**（先分库） |
| 分类第二维 | 学科 | 形态（术语/思维模型/概念） |
| 分类第三维 | — | 学科 |
| PREFIX | TC / CC / MM | 已证实 TC/TM/TN · 原创 OT/OM/OC |
| 注册表 | 无 library 字段 | **新增 library: atomic/original** |

**一句话**：分类逻辑从「按形态分」升级为「**先分已证实/原创，再分形态，再分学科**」，三层更清晰、条目多也不乱。

## 二、PREFIX 速记

**已证实（T = Trusted/Tested）：**
- `TC` = 已证实术语（Term Card）
- `TM` = 已证实思维模型（Model）
- `TN` = 已证实概念（coNcept）

**原创（O = Original）：**
- `OT` = 原创术语
- `OM` = 原创思维模型
- `OC` = 原创概念

## 三、4 个 Python 脚本要改什么

> 你的脚本靠「解析文件名 PREFIX」和「注册表」工作。PREFIX 体系变了，以下要同步改。改完先 dry-run 验证，别直接 --apply。

### kn_common.py（共享库）
- **PREFIX 解析**：原来识别 `TC/CC/MM`，改为识别 6 种 `TC/TM/TN/OT/OM/OC`。
- **新增判断函数**：根据 PREFIX 首字母判断 library（`T*`→atomic，`O*`→original）。
- **扫描路径**：新增扫 `04-原创/`，原来只扫 `03-Atomic/`。

### kn_dedup.py（去重 + 注册表）
- **全库搜索范围**：从「只搜 03-Atomic」扩展到「03-Atomic + 04-原创」。
- **新增边界检测**：同名概念若**一个在 atomic、一个在 original** → 不自动合并，回报用户（这是 v2.5 最重要的新去重规则）。
- **注册表写入**：新增 `library` 和 `form` 字段的读写。
- **类型判断**：先判 library（已证实/原创），再判 form（术语/模型/概念），两级决策树。

### kn_status.py（状态机）
- **扫描范围**：`@知识图谱` 的双链检测，范围加上 `04-原创/`。
- 其余状态流转逻辑不变。

### kn_links.py（幽灵链接）
- **链接解析**：识别新的 6 种 PREFIX 开头的 `[[...]]` 链接。
- 修复逻辑不变。

### .knowledge-network-config.yaml
- 已在 v2.5 模板里加了 `original:` 路径和顺延后的编号，直接用新模板。

## 五、如果你已有 v2.4 的旧笔记要迁移

旧库里 `03-Atomic/CC-*` 和 `MM-*`（原创的）要挪到新的 `04-原创/`：

- `CC-XX-概念名`（is_original: true）→ `04-原创/OC-原创概念/OC-XX-学科/`，文件名改 `OC-XX-概念名`

- `TC-*` 已证实术语 → 留在 03-Atomic 的 `TC-术语/`，PREFIX 不变
